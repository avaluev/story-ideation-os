🔄 10-Phase Pipeline Architecture

The pipeline executes deterministically from research through investor presentation, with quality gates at every critical juncture:

Phase 0: Intake & Validation
Parse theme input, classify content type, extract structured metadata. Gate: Research brief completeness ≥60%.
concept-drafter agent
Phase 1: Research (Parallel)
Genre saturation (market position), competitor comps (8+ tier-1 films), audience URLs. Gate: Citation coverage on all numeric claims.
concept-researcher agent
Phase 2: Draft v0
Generate 4-section investor template with framework labels (internal only). Gate: Structure validation.
concept-drafter agent
Phase 3: Challenge Loop (L1)
Adversarial P0/P1 interrogation. Max 3 patch rounds before reject. Gate: Kill-switch pass rate ≥70%.
concept-challenger agent Plateau-checked: stops on no improvement
Phase 4: Amplification Loop (L2)
Compound 29-vector TAM/SAM/SOM multiplier engine. Max 5 iterations OR stops on <5% delta. Gate: SOM ≥$100M.
audience-amplifier agent Plateau-detection enabled
Phase 5: Genius Audit (L3)
C001–C007 kill-switch framework (originality, cultural fit, franchise potential). Max 3 patch rounds. Gate: Min 2/7 passed.
genius-auditor agent
Phase 6: Consistency Check (L4)
Cross-phase drift detection (Python). Resolve conflicts: Python-computed > cited > research > assumptions. Gate: Drift score ≤0.15.
consistency-checker agent
Phase 7: Evaluation Gate
Tier-1 (SOM, citation count) + Tier-2 (Flesch-Kincaid, cliché score) validation.
Automated scoring
Phase 8: Narrator Redo Loop (L5)
Convert to investor markdown, strip internal IDs, apply style guide. Max 2 redo rounds on eval failure. Gate: No internal IDs detected.
concept-narrator agent
Phase 9: Output & Lessons
Final deliverable named after film title (not iter-N.md). Append insights to global lessons.jsonl.
Output filtering enforced

⚡ Scalability & Capability Drivers

1. Plateau-Checked Recursive Loops

Instead of fixed iteration counts, the system detects convergence and exits early:

Challenge Loop (L1)
≤3 patches
Amplification Loop (L2)
5 iters OR <5% Δ plateau
Genius Loop (L3)
≤3 patches
Consistency Loop (L4)
≤3 patches
Narrator Redo (L5)
≤2 rounds

Result: Average run time 30–90 minutes. No runaway feedback loops. Deterministic cost per concept.

2. Pure-Python Scoring (ADR-0002)

All numeric scores computed in pipeline/scoring.py, never by LLMs. Benefits:

  • Auditable: Every number is equation, not opaque model output
  • Reproducible: Same input → same score, always
  • Fast: Python execution (ms) vs LLM inference (seconds)
  • Cost-controlled: Scoring doesn't consume model quota

3. Compound Amplification Vectors (29)

The system detects synergies between audience multipliers. Example:

  • A-Grade Cast alone: 1.4x audience
  • Prestige Director alone: 1.5x
  • Both together: 1.4 × 1.5 × 1.3 (synergy bonus) = 2.73x

Non-linear multiplication vs additive math → higher-SOM concepts emerge from better vector combinations.

4. Resumable State Management (ADR-0001)

Every phase outputs atomic JSON. If interrupted:

  • System reads project state file
  • Resumes from next incomplete phase
  • Zero re-work on previously-completed phases
  • Enables long-running batch jobs and graceful interruption

5. Parallel Phase Execution

Phase 1 research agents (market, competitive, audience) run concurrently via Task fan-out. Reduces total time by ~40%.

6. Automatic Internal-ID Stripping (ADR-0010)

Before output, template filter removes all framework names, abbreviations, run IDs. Zero manual post-processing required.

✅ 6-Point Quality Gate System

Every phase transition enforced by automated validation:

Gate 0: Research Completeness

Triggers: After intake.

Pass Criteria: completeness_score ≥60%, content_type identified, target geo populated.

Fail Action: Surface clarification questions to user.

Gate 1: Citation Coverage

Triggers: After Phase 1 agents.

Pass Criteria: 100% numeric claims have _source fields.

Fail Action: Re-invoke agent with missing field list.

Gate 2: Math Integrity

Triggers: After financial modeling.

Pass Criteria: All scores from executed Python, no LLM fabrication.

Fail Action: Halt, display errors, require confirmation.

Gate 3: Red Team Minimum

Triggers: After adversarial review.

Pass Criteria: ≥3 HIGH-severity issues identified.

Fail Action: Re-invoke with stricter instructions.

Gate 4: Regulatory Coverage

Triggers: After risk assessment.

Pass Criteria: Risk entry for every target geography.

Fail Action: Retry with missing geos listed.

Gate 5: SOM Threshold

Triggers: After amplification.

Pass Criteria: SOM ≥ $100M (hard gate).

Fail Action: Retry amplification loop or accept as "early-stage".

💡 Why This System Scales

1. No Manual Post-Processing

Output filtering is automated. No human needed to strip internal IDs, translate framework labels, or fix formatting. Run → Deliverable in one step.

2. Deterministic Cost Per Concept

Plateau detection caps iterations. You know upfront: $4–18 per concept, 30–90 min turnaround. No runaway inference costs.

3. Resumable Execution

Interrupted mid-run? Resume from the last completed phase. Zero wasted computation on retries.

4. Auditable Scores

All numbers come from Python equations, not LLM outputs. Transparency → investor confidence. Reproducibility → no arguing about SOM calculations.

5. Knowledge Base Separation

Dictionaries are static JSON files, not embedded in prompts. Versioning, updates, and A/B testing become trivial. Swap in a different cultural_moment_2026.json and re-run.

6. Plateau-Checked Loops

Instead of fixed iteration caps, the system measures convergence and exits early. Most challenging concepts resolve in 1–2 L1 patch rounds, not 3.

7. Parallel Research Execution

Market, competitive, and audience research agents run concurrently. Saves ~40% of total pipeline time.
